# Решение проблемы с подключением тестовой системы

## Диагностика

✅ База данных `distcomp` существует  
✅ Схема `distcomp` создана  
✅ Все таблицы созданы  
✅ Подключение с паролем `1902` работает  
❌ Подключение с паролем `postgres` НЕ работает  
❌ Тестовая система не может подключиться

## Проблема

Тестовая система пытается подключиться с паролем `postgres` (дефолтный), но в вашей системе пароль для пользователя `postgres` установлен как `1902`.

## Решения

### Решение 1: Изменить пароль PostgreSQL на 'postgres' (БЫСТРОЕ)

**Шаг 1:** Изменить пароль в PostgreSQL:
```powershell
.\change-password-to-postgres.ps1
```

**Шаг 2:** Изменить пароль в `application.properties`:
```properties
spring.datasource.password=postgres
```

**Шаг 3:** Перезапустить приложение

**Плюсы:** Быстро решает проблему  
**Минусы:** Меняет пароль системы

### Решение 2: Настроить тестовую систему на пароль '1902' (РЕКОМЕНДУЕТСЯ)

Найдите конфигурационный файл тестовой системы и установите:
```
Password: 1902
```

**Плюсы:** Не меняет настройки системы  
**Минусы:** Требует доступа к настройкам тестовой системы

### Решение 3: Использовать профиль 'test' (АЛЬТЕРНАТИВА)

Я создал файл `application-test.properties` с паролем `postgres`.

**Шаг 1:** Изменить пароль PostgreSQL:
```powershell
.\change-password-to-postgres.ps1
```

**Шаг 2:** Запустить приложение с профилем test:
```bash
--spring.profiles.active=test
```

## Рекомендация

**Лучший вариант:** Настроить тестовую систему на использование пароля `1902`.

Если это невозможно, используйте Решение 1 для быстрого исправления.

## Проверка после настройки

После применения решения, проверьте подключение:
```powershell
.\test-connection.ps1
```

Все тесты должны пройти успешно.
